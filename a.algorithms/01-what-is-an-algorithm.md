# What Is an Algorithm

## Overview

An algorithm is a **step-by-step set of instructions** used to solve a problem or complete a task. Every algorithm follows a logical sequence of actions that transforms an input into the desired output.

Algorithms are the foundation of computer programming. Before writing code, software engineers first design an algorithm that explains *how* the problem will be solved.

---

## Why It Matters

Algorithms are important because they:

- Solve problems systematically
- Improve software efficiency
- Reduce errors
- Help organize complex tasks
- Form the foundation of every computer program

Every application you use—from Google Search to YouTube and Google Maps—relies on algorithms.

---

## How an Algorithm Works

An algorithm generally follows three stages:

1. **Input** – Receive data or information.
2. **Process** – Perform a series of logical steps.
3. **Output** – Produce the final result.

Example:

```text
Input

↓

Process

↓

Output
```

---

## Everyday Examples

Algorithms are not limited to computers. We follow algorithms every day.

### Example: Making a Sandwich

1. Take two slices of bread.
2. Add your filling.
3. Place the second slice on top.
4. Cut the sandwich.
5. Serve.

Following these steps in order produces the expected result.

---

### Example: Following GPS Directions

Google Maps calculates an algorithm to determine the best route.

Example:

```text
Start

↓

Find Available Roads

↓

Choose Shortest or Fastest Route

↓

Reach Destination
```

## Algorithms in Programming

Every program is built around one or more algorithms.

Examples include:

- Searching for a file
- Sorting a list of numbers
- Logging a user into a website
- Processing online payments
- Sending emails
- Displaying search results

The code simply implements the algorithm.

---

## Characteristics of a Good Algorithm

A good algorithm should be:

### Correct

It should always produce the correct result.

---

### Efficient

It should solve the problem using as little time and memory as possible.

---

### Clear

Its steps should be easy to understand and follow.

---

### Finite

It must eventually stop after completing its task.

An algorithm that never ends is not useful.

---

## Example: Finding the Largest Number

Problem:

Find the largest number in a list.

Algorithm:

1. Assume the first number is the largest.
2. Compare it with the next number.
3. If the next number is larger, update the largest value.
4. Repeat until every number has been checked.
5. Display the largest number.

Notice that the algorithm explains the solution without using any programming language.

## Algorithms vs Programs

| Algorithm | Program |
|-----------|----------|
| A step-by-step solution | The actual implementation of the solution |
| Focuses on solving the problem | Focuses on executing the solution |
| Independent of programming language | Written in a programming language |
| Can be written as pseudocode or a flowchart | Written using code |

An algorithm comes **before** writing a program.

---

## Common Mistakes Beginners Make

Beginners often:

- Think algorithms only exist in programming.
- Confuse an algorithm with programming code.
- Write code before designing the algorithm.
- Focus on syntax instead of problem-solving.
- Assume there is only one algorithm for every problem.

Different algorithms can solve the same problem with different levels of efficiency.

---

## Key Takeaways

- An algorithm is a step-by-step procedure for solving a problem.
- Every computer program is built on one or more algorithms.
- A good algorithm is correct, efficient, clear, and finite.
- Algorithms are used in countless real-world applications.
- Designing the algorithm before writing code leads to better software.

---

## Summary

Algorithms are the foundation of computer science and software engineering. They provide a structured way to solve problems by breaking them into logical, ordered steps. Whether searching the web, navigating with GPS, processing payments, or sorting data, algorithms make modern computing possible. Learning algorithms is essential for writing efficient software and becoming a skilled software engineer.
